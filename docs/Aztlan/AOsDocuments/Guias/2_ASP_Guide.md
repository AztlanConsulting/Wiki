## Architecture Starter Pack Guide

Este documento sirve para saber qué pasos seguir para llenar el ASP (Architecture Starter Pack) para cada uno de los proyectos.

### Antes de empezar
Antes de empezar el ASP, verificar que no se tenga uno previo por proyecto. Tener la plantilla del ASP para basarse en dicho documento. Y, en caso de ser posible, reunir al equipo para que todos aporten opiniones durante el desarrollo del documento.

### Durante el desarrollo del ASP
- Basándose en la plantilla, cambiar el título del documento.
- Realizar un pequeño análisis en donde se describa el contexto del proyecto.
- Encontrar los objetivos a cumplir con el proyecto.
- Realizar un mapa del contexto, y en base a eso hallar los usuarios que usarían el sistema.
- Analizar el contexto para encontrar posibles supuestos (Ej.: se haría una página web, se haría una aplicación móvil).
- Elaborar un diagrama de componentes para encontrar y redactar los módulos principales en el proyecto.
- Estudiar lo realizado hasta el momento para hallar los datos sensibles que se tendrán que encriptar.
- En base al contexto y preferencias del cliente, analizar los atributos de calidad (requerimientos no funcionales) más importantes a aplicar.
- Ubicar los posibles riesgos que amenazan el éxito del proyecto, al tomar en cuenta el contexto de la situación (Ej.: datos sensibles a encriptar, gran cantidad de datos a almacenar).
- Definir las tecnologías a utilizar con base en el valor que aportan al proyecto y el nivel de dominio del equipo de desarrollo.
- Encontrar qué integraciones con servicios de terceros podrían aportar valor al proyecto.

### Después de redactar el documento
Con el ASP terminado, usarlo para ayudar a la planeación y detallado de requerimientos de software.
