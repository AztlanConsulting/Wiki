## Propósito
Establecer lineamientos técnicos formales para análisis, diseño, implementación y despliegue del software.

## Áreas de proceso y prácticas en las que se aplica
- REQM - Requirements Managment
- TS - Technical Solution
- VER - Verification
- CM - Configuration Management

## Política

1. El PO deberá asistir con al menos un miembro adicional a reuniones con el socio para la validación del avance del proyecto.
2. Todo diseño requiere aprobación del AO y un miembro adicional del equipo.
3. Si existe un desacuerdo técnico y tras dos intentos no se ha logrado solucionar:
- La Comunidad de Práctica decide.
- La decisión es vinculante.
4. Se establece hora límite para push.
5. Extensión de horario grupal requiere:
- Aviso 24 horas previas.
- Votación unánime.
- Una duración máxima de 3 horas extra.
