_v1.0 

## Propósito
Establecer mecanismos formales de control de calidad para garantizar productos funcionales, estables y confiables.

## Áreas de proceso y prácticas en las que se aplica
- PPQA - Process and Product Quality Assurance
- VER - Verification
- VAL - Validation

## Política

1. Se conformará un equipo rotativo de QA.
2. Toda funcionalidad:
- Requiere pruebas automatizadas (si aplica).
- No puede integrarse si rompe las pruebas existentes.
3. Cada prueba debe validarse por **mínimo** dos miembros.
4. Se implementará Code Freeze progresivo:
- 1 semana (Periodo 1)
- 2 semanas (Periodo 2)
- 3 semanas (Periodo 3)
5. Errores posteriores a la fecha de entrega:
- No se modifica el código.
- Se documenta y comunica formalmente al socio.
