# Guía para asignar un nivel de necesidad o dificultad de skills

## Propósito
Establecer una guía para la definición correcta de los niveles necesarios y de dificultad de las skills del proyecto.

## Desarrollo

### Nivel de Necesidad
| Nivel | Descripción     | Evidencia                                                  |
| ----- | --------------- | ---------------------------------------------------------- |
| 0     | No conocimiento | No ha trabajado con la tecnología                          |
| 1     | Básico          | Conoce conceptos y puede seguir documentación              |
| 2     | Intermedio      | Puede implementar funcionalidades simples                  |
| 3     | Avanzado        | Puede diseñar soluciones con la tecnología y mentorizar    |

### Nivel de prioridad
| Prioridad | Descripción | Cobertura recomendada del equipo |
| --------- | ----------- | -------------------------------- |
| Crítica | Habilidad indispensable para implementar funcionalidades principales o arquitectura. | 80 - 100% |
| Alta | Habilidad necesaria para implementar componentes importantes del sistema. | 60 - 80% |
| Media | Habilidad útil para mejorar eficiencia, calidad o mantenibilidad. | 30 - 60% |
| Baja | Habilidad complementaria o especializada para optimizaciones o funcionalidades secundarias. | 0 - 30% |

### Nivel de Dificultad
| Dificultad | Descripción                       |
| ---------- | --------------------------------- |
| Baja       | Puede aprenderse en < 2 días      |
| Media      | Requiere una semana de estudio    |
| Alta       | Requiere experiencia previa       |
| Muy Alta   | Difícil de aprender en el periodo del proyecto |
