# Training Plan para Proyectos

## Propósito

Definir las acciones necesarias para cubrir brechas de conocimiento detectadas en el equipo del proyecto, asegurando que las habilidades requeridas se encuentren disponibles para la ejecución exitosa del proyecto.


## Habilidades con Brechas Detectadas

Identifica en el inventario de Skills aquellas habilidades que no cubran la cobertura deseada, consulta la [guía de niveles de Skill](../Guías/GUI14_Guía_Nivel_De_Skills.md) para esto.

## Plan de Formación

Para cada skill no cubierta, se definen las acciones que permitirán adquirir o fortalecer las habilidades.

Dependiendo del porcentaje de cobertura, se deberá definir los miembros del equipo que tomarán una estrategia de aprendizaje, como mínimo se debe llegar al límite inferior sugerido por la guía de niveles de Skill.

## Aprendizaje

Cada miembro del equipo asignado deberá seguir estos pasos.

| Fases | Actividades |
|--------------|--------------|
| Consulta del inventario organizacional | Consulta la [Base de Skills](https://docs.google.com/spreadsheets/d/1fTEIn50jTNEAErV28CrP1KxcjjsE_eJUXX-Y_yDiCIM/edit?gid=0#gid=0) para verificar si la habilidad requerida ya existe en la organización. |
| Revisión de documentación y expertos | Identifica personas de consulta y revisa la documentación oficial e interna disponible (Spikes, ADRs). |
| Evaluación de cobertura | Determina si la habilidad está al menos parcialmente cubierta (documentación formal e interna). En caso de ser necesario, asigna un responsable para su documentación conforme a las guías de spikes o ADRs. |
| Estrategia de adquisición de conocimiento | Define el método de adquisición de conocimiento (capacitación, autoestudio, mentoría interna, spike o investigación). |
| Formalización del aprendizaje | Documenta los hallazgos en un resumen personal. Genera un ADR o, en relación al nivel de aprendizaje adquirido, registra personas de consulta. |

## Estrategias de Aprendizaje

Las siguientes estrategias pueden utilizarse para adquirir conocimiento técnico:

- **Autoestudio** mediante documentación oficial.
- **Spike o investigación** para documentar los hallazgos.
- **Mentoría interna** con miembros con mayor experiencia o personas de consulta.
- **Capacitación** con un profesor o mentor.

## Evidencia de Aprendizaje

Si el equipo lo considera necesario, cada miembro podrá documentar su aprendizaje para validar que la habilidad fue adquirida, se puede presentar alguna de las siguientes evidencias:

- Spike técnico documentado.
- ADR relacionado con la tecnología.
- Documento técnico o guía interna.
- Demostración funcional dentro del proyecto con un PR aprobada.

## Seguimiento

El seguimiento al plan de skills deberá darse cada que ocurra alguna de estas situaciones:
- Cada que termine un periodo establecido (ej. sprints) 
- Cada que una nueva habilidad sea identificada y añadida al inventario de skills. 
- Si el AO cree necesario reconsiderar el nivel de una skill, de igual manera se repite el paso.
