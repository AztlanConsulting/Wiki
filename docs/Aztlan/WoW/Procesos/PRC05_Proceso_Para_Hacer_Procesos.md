## Propósito
Definir una serie de pasos para crear nuevos procesos que proporcionen al departamento una forma clara y entendible de trabajo.

## Entradas
Prácticas del departamento:
- Problemas o necesidades documentados en minutas del departamento.
Objetivo:
- Información recopilada entre miembros del departamento acerca de lo que se quiera lograr con el proceso.

## Descripción
| Fases | Actividades | Prácticas asociadas al CMMI |
|--------------|--------------|--------------|
| Entendimiento | Identificar un área de mejora en la forma de trabajo del departamento |    |
| Entendimiento | Verificar si ya hay un proceso definido que aborde la necesidad identificada |    | 
| Entendimiento | Si ya existe un proceso, validar si se le da un uso adecuado y si aborda correctamente la necesidad |    | 
| Ejecución |  Redactar el proceso siguiendo la plantilla de procesos [PLA02 - Plantilla de Procesos](../Plantillas/PLA02-Proceso.md).|    |  
| Ejecución | Validar con el SEPG las áreas del CMMI ligadas al proceso. |    |  
| Ejecución | Seguir el estándar de creación de procesos |    |  
| Validación | Validar que el proceso siga las políticas establecida |    |  
| Validación | Solicitar a un miembro del departamento que revise el proceso y seguir la retroalimentación otorgada|    |  
| Documentación | Añadir el proceso a la Wiki y al mapa de procesos |    |  

## Salidas
Un nuevo proceso documentado en el mapa de procesos y wiki.
