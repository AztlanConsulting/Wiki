## Propósito

Definir el objetivo y los elementos de trabajo dentro del Sprint, priorizando, estimando y 
asignando el trabajo de este Sprint

## Entradas

- Backlog (PVG)
- WBS
- Velocidad del equipo
- Definición de DONE

## Descripción

| Pasos         | Actividades                                                                                                            | Prácticas Asociadas al CMMI       |
|---------------|------------------------------------------------------------------------------------------------------------------------|-----------------------------------|
| Revisión      | El TL revisa con el equipo los planes que afectan al proyecto (PVG, WBS, tareas pendientes, objetivo del sprint anterior) para asegurar que todos comprendarn el estado actual del plan del proyecto                                                                                             | PP.3.1 |                                       
| Definición    | Establecer el [objetivo del Sprint](../Guías/Generar_Objetivo_Sprint)                                                  | RM.1.1 PP.3.1                     |
| Definición    | El TL tomá las actividades del WBS y las desgloza en el [PVG](https://docs.google.com/spreadsheets/d/1kg_PnTUw6GnHll7XnYxqXmGK9umSope4nmvp1Qzt5Nc/edit?usp=sharing)                                                                                                                             | PP.1.1 PP.2.1                     |
| Definición    | El PO lee la Definición de DONE al equipo                                                                              | RM.1.5                            |
| Definición    | Obtener el presupuesto total del equipo en horas                                                                       | PP.1.2 PP.2.4 RM.1.2              |
| Definición    | El TL selecciona un itema de trabajo y lo presenta al equipo para ralizar la estimación                                | PP.1.2                            |
| Definición    | El equipo vota sobre la comlejidad que tiene el item seleccionado y se le asigna un story point de acuerdo a la [Plantilla para convertir story points a horas](../Plantillas/plantilla_StoryP.md)                                                                                                          | PP.1.4 PP.1.2                     |
| Definición    | El equipo asigna el mejor caso, peor caso y caso promedio de story points en el [PVG](https://docs.google.com/spreadsheets/d/1kg_PnTUw6GnHll7XnYxqXmGK9umSope4nmvp1Qzt5Nc/edit?usp=sharing) y se hace el calculo del costo en horas que tendra la tarea               | PP.1.4                            |
| Definición    | El equipo compara las horas disponibles del Sprint con las tareas planeadas para asegurar que el plan sea realista     | PP.3.2                            |
| Definición    | Si es necesario, hacer [Backlog Grooming](./PRC07_Proceso_Para_Grooming)                                               | PP.1.2 RM.1.1 RM.1.2              |
| Definición    | El equipo asigna una prioridad a cada tarea de acuerdo a las necesidades del proyecto y de los stakeholders            | PP.2.1                            |
| Preparación   | Añadir fecha planeada para cada tarea de acuerdo a la prioridad                                                        | PP.2.1 PP.2.7                     |
| Preparación   | Los Team Members eligen las tareas del Backlog acorde con la [velocidad del equipo](../Guías/Obtener_Velocidad_Equipo) | PP.2.1                            |
| Ejecución     | Establecer el responsable para cada tarea                                                                              | PP.2.1                            |
| Documentación | Actualizar el Backlog con todas las tareas asignadas y estimadas con fechas                                            | PP1.2 PP.2.7 PP.3.3 RM.1.2 RM.1.4 |

## Salidas

- PVG actualizado
- Sprint Backlog estimado
- Objetivo del Sprint
