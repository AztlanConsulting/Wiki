# Proceso para planificar recursos del proyecto
_v1.0 /_ PP 2.4

## Propósito
Definir los recursos y costo del proyecto en base a su nececidades

## Entradas
- Análisis recursos del proyecto 
- Plan del proyecto
- Roles departamentales 
- Requerimientos tecnicos del Proyecto


## Descripción
| Fase | Actividades | Prácticas CMMI |
|------|------------|----------------|
| Identificación y Registro | Definir y registrar Recursos del equipo, Los recursos pueden ser Tecnicos, Humanos, Infrestructura, Conocimientos  Y Heramientas | | 
| Definir Heramientas| Establecer las heramientas que los integrantes asi como los conocimientos que posean que esten integrados en el plan | PP 2.5 |
| Analisis de Costos | Identificar los costos de las heramientas | PP 2.7 |
| Adquicicion y Configuracion del proyecto | Definir Dominios, APIs y SSLs del proyecto | PP 2.7 |
| Monitoreo y control | Revisaual constantemente el consumo de las heramientas con costos (APIs) no exedan lo planiado y sus certificados no venzcan | PP 2.7 |
| Monitoreo y control | Revisaual constantemente el consumo de las heramientas con costos (APIs) no exedan lo planiado y sus certificados no venzcan | PP 2.7 |
| Planificacion Seguridad  | Definir roles de permisos de los Recusos con costo | PP 2.7 |

## Salidas
- Plan de Recursos
- Matriz de Conocimiento necesarios para el proyecto
